Чтобы переименовать ветку `master` в `main` в репозитории GitHub, выполните следующие шаги:


### 1. Переименование локальной ветки

Перейдите в каталог вашего локального репозитория и выполните:

```bash
git branch -m master main
```

Это переименует локальную ветку `master` в `main`.

### 2. Обновление удалённой ветки

Отправьте переименованную ветку на сервер:

```bash
git push -u origin main
```

Теперь на GitHub существует ветка `main`, но ветка `master` пока остаётся.


### 3. Смена ветки по умолчанию на GitHub

1. Откройте ваш репозиторий на GitHub.
2. Перейдите в **Settings** → **Branches**.
3. В разделе **Default branch** нажмите на значок карандаша (или кнопку редактирования).
4. Выберите `main` из списка веток.
5. Нажмите **Update**.

### 4. Удаление старой ветки `master` на сервере

После смены ветки по умолчанию удалите старую ветку `master`:

```bash
git push origin --delete master
```

### 5. Обновление локальных репозиториев у других участников

Если другие разработчики используют этот репозиторий, им нужно выполнить следующие команды:

```bash
# Переключиться на master
git checkout master

# Переименовать в main
git branch -m master main

# Получить обновления
git fetch

# Удалить связь с origin/master
git branch --unset-upstream

# Установить связь с origin/main
git branch -u origin/main


# Обновить символическую ссылку
git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main
```

### Важные замечания

- **Документация и ссылки.** Проверьте, нет ли ссылок на `master` в документации, README или внутренних ссылках (например, `https://github.com/.../blob/master/...`).
- **CI/CD и действия GitHub.** Обновите все workflow‑файлы (GitHub Actions), где упоминается `master`.
- **Зависимости пакетов.** Если ваш репозиторий используется как зависимость (например, в npm, Composer), убедитесь, что сторонние проекты не ссылаются на `master`. Возможно, стоит временно оставить `master` как алиас.

После выполнения всех шагов ветка `master` будет полностью заменена на `main` как в локальном, так и в удалённом репозитории.
